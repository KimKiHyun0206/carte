# 들어가기 앞서서

여태까지 자바의 문법이나 객체지향 언어의 디자인 패턴에만 관심이 많았다. 하지만 이번에 자바란 어떤 언어인지 제대로 짚고 넘어가려고 한다

# Java

> 웹 애플리케이션 코딩에 널리 사용되는 프로그래밍 언어이다

* 자바는 그 자체로 플랫폼으로 사용할 수 있는 다중 플랫폼, 객체 지향 및 네트워크 중심 언어이다
* 모바일 앱 및 엔터프라이즈 소프트웨어에서 빅 데이터 애플리케이션 및 서버 측 기술에 이르기까지 모든 것을 코딩할 수 있다
* 빠르고, 안전하며, 안정적인 프로그래밍 언어이다

## Java : 용도

> 지역화된 분산 소프트웨어를 구축한다

### 1. 게임 개발

다양한 모바일, 컴퓨터 및 비디오 게임에 Java로 만들어진다.
기계 학습이나 가상 현실과 같은 첨단 기술이 통합된 최신 게임도 마찬가지이다.

### 2. 클라우드 컴퓨팅

자바는 WORA(Write Once and Run Anywhere)라고도 하며 분산 클라우드 기반 애플리케이션에 적합하다.
클라우드 공급자는 다양한 기본 플랫폼에서 프로그램을 실행하기 위해 자바를 선택한다.

### 3. 빅 데이터

자바는 복잡한 데이터 집합과 방대한 양의 실시간 데이터를 처리할 수 있는 데이터 처리 엔진이 사용된다.

### 4. 인공 지능

자바는 기계 학습 라이브러리의 발전소 역할을 한다.
자바는 자연어 처리 및 딥 러닝과 같은 인공 지능 애플리케이션 개발에 적합한 안정성과 속도를 갖췄다.

### 5. 사물 인터넷

자바는 인터넷에 독립적으로 연결할 수 있는 엣지 디바이스의 센서 및 하드웨어 프로그래밍에 사용되었다.

## Java : 인기 있는 이유

> Java는 사용하기 쉽도록 설계되었기 때문이다.

### 고품질 학습 리소스

* 자바는 오랫동안 사용되어 왔기 때문에 새로운 프로그래머가 사용할 수 있는 리소스가 많이 남아있다
* 개발자가 학습 과정 내내 사용할 수 있는 책이나 예제가 많이 있다

### 기본 제공 함수 및 라이브러리

* 자바는 개발자가 처음부터 모든 기능을 작성할 필요가 없다
* 자바는 다양한 라이브러리로 개발 편의성을 가지고 있다
    * Math, Collection 등

### 적극적인 커뮤니티 지원

* 자바에는 코딩 문제 발생 시 개발자를 지원할 수 있는 많은 사용자와 커뮤니티가 있다
* 자바 플랫폼 소프트웨어도 정기적으로 유지 관리되고 업데이트 된다

### 고품질 개발 도구

자바는 자동화 편집, 디버깅, 데스트, 배포 및 변경 관리를 지원하는 다양한 도구를 제공한다

### 플랫폼 독립적

* 자바 코드는 다시 작성하지 않고도 Windows, Linus, iOS 또는 Android와 같은 기본 플랫폼에서 실행할 수 있다.
* 여러 디바이스에서 애플리케이션을 실행하려는 오늘날의 환경에 특히 더 유용하다

### 보안

* 사용자는 네트워크를 통해 신뢰할 수 있는 자바 코드를 다운로드하고 해당 자바 코드가 해를 끼칠 수 없는 안전한 환경에서 이를 실행할 수 있다
* 신뢰할 수 없는 코드가 호스트 시스템을 바이러스에 감염시킬 수 없으며 하드드라이브에서 파일을 읽거나 쓸 수 없다
* 자바의 보안 수준 및 제한 사항도 고도로 구성 가능하다

## Java : 특징

### 자바는 객제지향 언어이다

* 객체지향 프로그래밍(OOP : Object oriented Programming)이란 프로그램을 개발하는 기법으로 부품에 해당하는 객체들을 먼저 만들고, 부품들을 조립 및 연결하여 전체 프로그램을 완성하는 것을
  말한다
* 객체를 만들기 위해서 설계도인 클래스를 작성하고, 객체와 객체를 연결하여 목적에 맞는 프로그램을 만들어낸다
* 객체지향 언어의 특징인 캡슐화, 상속성, 다형성을 완벽하게 지원한다

### 이식성이 높다

* 서로 다른 실행 환경을 가진 시스템 간에 프로그램을 옮겨 실행할 수 있는 것을 말한다
* 자바는 자바 실행환경(JRE)이 설치되어 있는 모든 운영체제에서 실행이 가능하다

### 인터프리터 언어이다

* 컴파일 언어인 동시에 인터프리터 언어이다
* 텍스트 소스를 컴파일하여 2인 파일(.class)로 만든 다음 자바 런타임이 클래스 파일을 인터프리트 하면서 실행한다
* 시스템에 무관한 2진 파일을 만듦으로써 자바는 컴파일 언어에 가까운 속도와 시스템 독립성을 얻을 수 있다

### 메모리를 자동으로 관리한다

* 다른 언어는 메모리에 생성된 객체를 제거하기 위해서 개발자가 직접 코드를 작성해야 한다. 만약 이 작업을 성실하게 해주지 않으면 프로그램은 불안정해지고 갑자기 다운될 수도 있다
* 자바는 개발자가 직접 메모리에 접근할 수 없으며 메모리는 자바가 직접 관리한다
* 객체 생성시 자동으로 메모리 영역을 찾아서 할당하고 사용이 완료되면 **Garbage Collector**를 실행시켜 자동으로 사용하지 않는 객체를 제거한다
* 덕분에 개발자는 메모리 관리의 수고를 덜고 핵심 기능 코딩에 집중할 수 있다

### 멀티 스레드 구현이 쉽다

운영체제에 따라 멀티 스레드를 구현하는 방법이 다르지만 자바는 스레드 생성 및 제어와 관련된 라이브러리 API를 제공하므로 운영체제에 상관없이 멀티 스레드를 쉽게 구현할 수 있다

### 동적 로딩을 지원한다

* 실행시에 모든 클래스가 로딩되는 것이 아니라 필요한 시점에 클래스를 로딩하려 사용할 수 있다는 장점을 지닌다
* 애플리케이션의 변경사항이 발생해도 비교적 적은 작업만으로도 처리할 수 있는 유연한 애플리케이션을 작성할 수 있다

### 분산환경을 지원한다

* 자바에는 TCP/IP 라이브러리가 기본적으로 포함되어있다
* 또한 HTTP 와 HTP 프로토콜을 기본적으로 지원한다

### 오픈소스 라이브러리가 풍부하다

* 자바는 오픈소스 언어이고 자바 프로그램에서 사용하는 라이브러리 또한 오픈소스의 양이 방대하다
* 고급 기능을 구현하는 코드를 작성하는 대신 검증된 오픈소스 라이브러리를 사용하면 애플리케이션을 다시 컴파일할 필요가 없다
* 따라서 유지보수를 쉽고 빠르게 진행할 수 있다

### 단점

* 실행을 위해 JVM을 거쳐야 하기 때문에 다른 언어에 비해 실행 속도가 느리다
* 자바는 예외 처리가 잘 되어 있지만 개발자가 하나씩 처리를 지정해야 한다는 불편함이 있다
* 자바는 다른 언어에 비해 작성해야 하는 코드의 길이가 긴 편이다

## Java : 동작 방법

> 모든 프로그래밍 언어는 시스템과 통신하는 수단이다. 기계 하드웨어는 전자 통신에만 응답한다. 자바와 같은 고급 프로그래밍 언어는 인간 언어와 하드웨어 언어 사이의 다리 역할을 한다. 자바를 사용하려면 개발자는
> 다음 두 가지를 이해해야 한다

### Java API

> 자바 플랫폼에서 번들로 제공되는 중요한 소프트웨어 구성 요소이다.

* 기존 기능을 자신의 코드에 플러그 앤 플레이할 수 있는 미리 작성된 자바 프로그램이다
* 예를 들어 수학 연산이나 텍스트를 조작할 수 있다
* 개발자가 작성한 모든 자바 애플리케이션 코드는 일반적으로 Java API 밒 Java 라이브러리의 신큐 코드와 기존 코드를 결합한다

### Java Virtual Machine

> 자바 플랫폼과 기본 머신 하드웨어 간의 추가 추상화 계층 역할을 한다

* 자바 소스 코드는 JVM이 설치된 시스템에서만 실핼할 수 있다
* JVM이 필요한 이유에 대한 답은 프로그래밍의 역사에서 찾을 수 있다

### 프로그래밍의 역사

> 자연 프로그래밍 언어가 처음 개발되었을 때 기본 하드웨어와 통신하는 방식에 따라 크게 두 가지 범주로 나뉘었다

#### 1. 컴파일러

* 완전한 프로그램은 컴파일러를 사용하여 자연스러운 영어와 유사한 구문으로 작성된 다음 언어가 전체 코드를 기계어로 컴파일하거나 번역한다
* 컴파일된 코드는 하드웨어에서 실행된다

#### 2. 인터프리터

* 인터프리터를 사용하면 모든 고급 코드 문장이 즉석에서 기계어로 해석된다
* 작성된 문장은 다음 문장을 보기 전에 하드웨어에서 즉시 실행된다

## Java : 런타임 환경

> Java프로그램은 JVM을 사용하여 위의 두 가지 방법으로 결합한 최초의 언어였다

* 자바 코드 컴파일러를 JVM이라고 한다
* 모든 자바 파일은 먼저 바이트 코드로 컴파일된다
* 자바 바이트코드는 JVM에서만 실행할 수 있다
* 그런 다음 JVM은 바이트코드를 해석하여 기본 하드웨어 플랫폼에서 실행한다
* 따라서 애플리케이션이 Windows인 경우 JVM은 이를 Windows용으로 해석한다
    * Linus와 같은 오픈 소스 플랫폼에서 실행 중인 경우 JVM은 이를 Linux용으로 해석한다
